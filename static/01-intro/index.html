Einführung in die Vorlesung

## Fachkonpetemz Die Studierenden kennen die grundlegenden Kernkonzepte der Nebenläufigkeit und Internetkommunikation sowie die darauf aufbauenden aktuellen Technologien und Werkzeuge für den Entwurf und für die Implementierung von Webanwendungen. Die Studierenden kennen die technischen Grundlagen, Konzepte, Architekturen und Technologien verteilter Systeme und Anwendungen sowie die gängigen Methoden, Werkzeuge, Frameworks und Entwurfsmuster für die Entwicklung verteilter Anwendungen. Die Studierenden können die Konzepte, Werkzeuge und Methoden der Webprogrammierung anwenden und autonom Webanwendungen entwerfen, implementieren und testen. ## Methodenkompetenz Die Studierenden können die Konzepte, Werkzeuge und Methoden der verteilten Systementwicklung anwenden und unter Einsatz von Frameworks und Entwurfsmustern einfache verteilte Anwendungen entwerfen, implementieren und testen. ## Personale und soziale Kompetenz Die Studierenden können eigenständig verteilte Anwendungsarchitekturen erarbeiten. Sie können stichhaltig und sachangemessen über Konzepte, eigene Entwürfe und deren Implementierungen sowie die damit verbundenen Probleme argumentieren, eigene Umsetzungen plausibel darstellen und eventuelle Fehler nachvollziehbar gegenüber anderen begründen. ## Übergreifende Handlungskompetenz Die Studierenden können eigenständig aktuelle Methoden der Analyse, des Entwurfs und der Implementierung verteilter Systeme und Anwendungen für betriebliche Problemstellungen diskutieren, bewerten und anwenden.

## Didaktisches Modell {.mt-0} **Präsenzvorlesung** mit zeitlich flexiblem **Selbststudium** sowie einem semesterbegleitenden **Programmierprojekt** zur praktischen Anwendung. Das Programmierprojekt wird gleichzeitig als Prüfungsleistung benotet.
## Benötigte Software {.mt-0} * Visual Studio Code * Git * Docker * Postman Anstelle von Visual Studio Code kann auch eine andere IDE verwendet werden, so lange diese ohne ein Projekt anzulegen Quellcodes bearbeiten kann. {.dhbw-optional} Anstelle von Postman kann auch ein anderes Werkzeug zum Testen von HTTP-Endpunkten wie Soap UI, CURL, HTTPy, ... verwendet werden. {.dhbw-optional}
## Rahmenbedingungen {.mt-0} * Bearbeitung in selbstgewählten Dreiergruppen * Jede Gruppe implementiert eine eigene Webanwendung * Die Bearbeitung erfolgt parallel zur Vorlesung * Abgabe am letzten Sonntag vor Beginn der Praxisphase

Einzureichende Artefakte

  1. Eine Beschreibung der zu erstellenden App
  2. Den vollständigen Quellcode der App, bestehend aus:
    • Link auf das GitHub-Repository
    • Ordnerstruktur mit allen Build Files
    • Definition der Backend-API im OpenAPI-Format
    • Quellcode zur Implementierung der API
    • Quellcode einer dazu passenden Single Page App*
    • Konfiguration des Gatewayservers
  3. Eine zusammenfassende Bewertung der durch die App umgesetzten Cloud-Prinzipien
